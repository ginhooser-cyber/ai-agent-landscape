# Contributing to AI Agent Landscape

Thanks for helping improve the map.

This repo is trying to be useful, not just large.
So the standard is simple: add signal, not noise.

## Good contributions
- strong new agent entries
- stale pricing or licensing fixes
- category corrections
- better practical descriptions
- official-source links
- cleaner distinctions between products and frameworks

## Weak contributions
- vague hype language
- duplicate tools with no new insight
- dead links
- feature claims without sources
- stuffing the list just to increase the count

## Entry format
Most entries live in `data/agents.csv`.

Use these fields:
- `name`
- `url`
- `category`
- `open_source`
- `best_for`
- `pricing`
- `hosting`
- `notes`

## Writing style
Write like a sharp operator.
Not like a landing page.

Good:
- `Strong for browser automation and repetitive web tasks.`
- `Good OSS option if you want a local coding workflow.`

Bad:
- `The ultimate revolutionary AI platform for all use cases.`

## Source policy
Prefer official sources whenever possible:
- official repo
- official docs
- official product page

If a claim is subjective, write it as a judgment call instead of a fact.

## Pull requests
Small focused PRs are ideal.
If you are adding many tools, group them by category.

## Issues
Use issues for:
- missing tools worth adding
- broken links
- wrong pricing or licensing info
- category disputes
- methodology feedback
