# Methodology

This repo is not trying to be a random giant list.

The goal is to make the AI agent landscape easier to navigate for real users.

## Inclusion rules
A tool is a good fit for this repo if it is at least one of the following:
- a widely used AI agent product
- a meaningful open-source agent project
- an agent framework used to build real systems
- a developer-facing tool that clearly fits agentic workflows

## What we track
When possible, entries should capture:
- category
- open source vs closed
- best-for use case
- pricing model
- hosting model: local, cloud, or hybrid
- practical notes on strengths and limits

## What we avoid
- pure marketing copy
- vague claims without a source
- dead projects unless they are historically important
- stuffing the repo with near-duplicates just to inflate the count

## Opinion policy
This repo is intentionally curated and opinionated.

That means:
- not every entry is equally recommended
- short notes may include practical judgment calls
- subjective claims should be written as subjective claims, not facts

## Update policy
- prioritize accuracy over speed
- prefer official sources when available
- welcome corrections via issues and pull requests
